How this works

This page drives the projector exactly like its own web UI: native HTML form posts to /tgi/status.tgi (cross-origin form posts need no CORS). JavaScript cannot read the projector's responses cross-origin, so live state is shown via the embedded pages on the right. Connect first — it loads the projector root in a hidden frame to establish the ATOP session cookie. Defaults below mirror a healthy idle unit; for fidelity, copy current values from the embedded status page before sending (the firmware applies the whole form, see docs/PROTOCOL.md).

Login (password-protected projectors only)

Username is fixed to administrator by the firmware. Open the login page on the right (tab "Login"), copy the value of the hidden Challenge field from view-source (or just type the password below and press Login — the challenge is fetched from a fresh frame load, then Response = md5("admin"+password+challenge) is posted to /tgi/login.tgi).

Leave Challenge empty only if you pasted it before it expired; each login needs a fresh one.
